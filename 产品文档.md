# Blog_mini 博客系统 - 产品文档

## 1. 产品概述

Blog_mini是一个开源的博客系统，使用Python开发完成，具有简洁的界面和强大的后台管理，用户可以轻松架设属于自己的个人博客网站。

### 1.1 产品定位

- **目标用户**：个人博客爱好者、技术博主、初学者学习参考
- **核心价值**：让每个人都轻松拥有可管理的个人博客
- **产品特点**：简洁易用、功能完整、易于扩展、部署简单

### 1.2 主要功能

| 功能模块 | 核心功能 |
|---------|----------|
| 文章管理 | 发布、编辑、删除文章，支持富文本编辑 |
| 分类管理 | 创建、编辑、删除分类，支持分类导航 |
| 评论系统 | 访客评论、回复评论、评论审核 |
| 导航管理 | 自定义导航菜单，支持排序和隐藏 |
| 博客配置 | 自定义博客标题、签名、主题 |
| 插件系统 | 支持插件扩展，内置博客统计功能 |
| 访问统计 | 记录博客访问量和文章阅读量 |

## 2. 系统架构

### 2.1 架构设计

Blog_mini采用了Flask框架的工厂模式设计，具有清晰的分层架构：

- **表示层**：Flask模板系统 + Bootstrap前端框架
- **业务逻辑层**：Flask视图函数 + 蓝图机制
- **数据访问层**：SQLAlchemy ORM框架
- **数据存储层**：SQLite/MySQL数据库

### 2.2 技术栈

| 类别 | 技术/框架 | 版本 |
|------|-----------|------|
| 后端框架 | Flask | 3.0.0 |
| ORM框架 | SQLAlchemy | 2.0.35 |
| 前端框架 | Bootstrap | 3.3.7 |
| 富文本编辑器 | TinyMCE | 4.0+ |
| 认证系统 | Flask-Login | 0.6.3 |
| 表单处理 | Flask-WTF | 1.2.1 |
| 数据库迁移 | Flask-Migrate | 4.0.7 |

## 3. 核心功能说明

### 3.1 文章管理

- **发布文章**：支持标题、内容、分类、来源、摘要等信息
- **编辑文章**：支持修改已发布的文章
- **删除文章**：支持删除文章
- **文章列表**：支持分页显示，按发布时间排序
- **文章搜索**：支持关键词搜索

### 3.2 分类管理

- **创建分类**：支持添加新的文章分类
- **编辑分类**：支持修改分类名称和介绍
- **删除分类**：支持删除分类
- **分类导航**：支持将分类关联到导航菜单
- **分类统计**：显示每个分类下的文章数量

### 3.3 评论系统

- **访客评论**：支持访客匿名评论
- **评论回复**：支持回复其他用户的评论
- **评论审核**：支持审核和禁用评论
- **评论分页**：支持分页显示评论

### 3.4 导航管理

- **创建导航**：支持添加自定义导航菜单
- **编辑导航**：支持修改导航名称和排序
- **删除导航**：支持删除导航菜单
- **导航关联**：支持将分类关联到导航

### 3.5 博客配置

- **基本信息**：修改博客标题、签名
- **主题设置**：修改导航栏样式

### 3.6 插件系统

- **系统插件**：内置博客统计插件
- **自定义插件**：支持添加自定义HTML插件
- **插件管理**：支持启用、禁用、排序插件

## 4. 部署说明

### 4.1 环境要求

- Python 3.6+
- pip包管理工具
- SQLite或MySQL数据库

### 4.2 本地部署

1. **克隆代码**：`git clone https://github.com/xpleaf/Blog_mini.git`
2. **安装依赖**：`pip install -r requirements.txt`
3. **初始化数据库**：`python manage.py db init && python manage.py db migrate && python manage.py db upgrade`
4. **添加初始数据**：`python manage.py deploy(product)`
5. **启动服务**：`python manage.py runserver`

### 4.3 服务器部署

- **Nginx+uWSGI**：推荐生产环境部署方案
- **Apache+mod_wsgi**：可选部署方案
- **Docker**：支持容器化部署

## 5. 扩展开发

### 5.1 插件开发

Blog_mini支持自定义插件，可以通过以下步骤开发插件：

1. 在后台管理中点击"插件管理"
2. 点击"添加插件"
3. 填写插件标题、说明和HTML内容
4. 保存插件

### 5.2 功能扩展

可以通过以下方式扩展系统功能：

1. 创建新的蓝图模块
2. 添加新的数据模型
3. 编写新的视图函数
4. 自定义模板
